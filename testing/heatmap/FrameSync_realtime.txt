## 1. Real-Time Processing Concepts

### The "Just-in-Time" Approach
If the STM32 can complete FFT processing faster than your video frame rate (e.g., < 33ms for 30 FPS), then:
- STM32 processes audio → FFT → beamforming → sends result to Pi
- Pi receives fresh data just as it needs to render the next frame
- No complex synchronization needed - just process the latest available data

### Key Advantages
- **Simpler implementation** - no timestamping, no ring buffers
- **Lower latency** - no buffering delays
- **Less memory usage** - no need to store multiple frames
- **Easier debugging** - straightforward data flow
- **Real-time by design** - if STM32 is fast enough, it's naturally synchronized

### Potential Issues
- **Timing dependency** - relies on STM32 being consistently fast
- **No tolerance for delays** - if STM32 is slow, you get stale data
- **No error recovery** - if one frame is late, it affects the next
- **Harder to optimize** - can't smooth out timing variations

```python
def monitored_realtime_pipeline():
    """
    Real-time pipeline with performance monitoring to detect timing issues.
    
    This version adds monitoring capabilities to help identify when
    the simple approach might not be sufficient.
    """
    
    # Configuration
    WIDTH = 640
    HEIGHT = 480
    FPS = 30
    ALPHA = 0.7
    
    # Setup camera
    camera = cv2.VideoCapture(0)
    if not camera.isOpened():
        raise RuntimeError("Failed to open camera")
        
    camera.set(cv2.CAP_PROP_FRAME_WIDTH, WIDTH)
    camera.set(cv2.CAP_PROP_FRAME_HEIGHT, HEIGHT)
    camera.set(cv2.CAP_PROP_FPS, FPS)
    camera.set(cv2.CAP_PROP_BUFFERSIZE, 1)
    
    # Setup FFmpeg
    ffmpeg_process = setup_ffmpeg_process(WIDTH, HEIGHT, FPS, "monitored_output.mp4")
    
    # Audio source setup
    # audio_source = setup_stm32_interface()
    
    # Performance monitoring
    frame_count = 0
    start_time = time.monotonic()
    frame_times = []
    processing_times = []
    
    print("Monitored Real-Time Heatmap Pipeline")
    print("Press 'q' to quit")
    
    try:
        while True:
            frame_start = time.monotonic()
            
            # Capture video frame
            ret, video_frame = camera.read()
            if not ret:
                print("Failed to capture video frame")
                continue
                
            # Get latest audio data
            audio_start = time.monotonic()
            # audio_data = audio_source.get_latest_fft_result()
            audio_data = generate_fake_heatmap(WIDTH, HEIGHT)
            audio_time = time.monotonic() - audio_start
            
            # Process
            process_start = time.monotonic()
            output_frame = apply_heatmap_overlay(audio_data, video_frame, ALPHA)
            process_time = time.monotonic() - process_start
            
            # Add performance info to frame
            elapsed = time.monotonic() - start_time
            fps_actual = frame_count / elapsed if elapsed > 0 else 0
            
            cv2.putText(output_frame, f"FPS: {fps_actual:.1f}", 
                       (10, 30), cv2.FONT_HERSHEY_SIMPLEX, 0.7, (255, 255, 255), 2)
            cv2.putText(output_frame, f"Audio: {audio_time*1000:.1f}ms", 
                       (10, 60), cv2.FONT_HERSHEY_SIMPLEX, 0.7, (255, 255, 255), 2)
            cv2.putText(output_frame, f"Process: {process_time*1000:.1f}ms", 
                       (10, 90), cv2.FONT_HERSHEY_SIMPLEX, 0.7, (255, 255, 255), 2)
            
            # Display and record
            cv2.imshow('Monitored Heatmap', output_frame)
            send_to_ffmpeg(output_frame, ffmpeg_process)
            
            # Record timing data
            frame_count += 1
            frame_times.append(time.monotonic() - frame_start)
            processing_times.append(audio_time + process_time)
            
            # Print status every 100 frames
            if frame_count % 100 == 0:
                avg_frame_time = np.mean(frame_times[-100:]) * 1000
                avg_process_time = np.mean(processing_times[-100:]) * 1000
                print(f"Frame {frame_count}: Avg frame time: {avg_frame_time:.1f}ms, "
                      f"Avg process time: {avg_process_time:.1f}ms, "
                      f"FPS: {fps_actual:.1f}")
                
                # Check for performance issues
                if avg_process_time > 33:  # More than 1 frame time
                    print(f"WARNING: Processing time ({avg_process_time:.1f}ms) exceeds frame time (33ms)")
                if fps_actual < FPS * 0.9:  # Less than 90% of target FPS
                    print(f"WARNING: FPS ({fps_actual:.1f}) below target ({FPS})")
            
            # Frame rate control
            if cv2.waitKey(1000 // FPS) & 0xFF == ord('q'):
                print("Quit requested by user")
                break
                
    except KeyboardInterrupt:
        print("\nPipeline interrupted by user")
    except Exception as e:
        print(f"Error in pipeline: {e}")
    finally:
        # Final statistics
        total_time = time.monotonic() - start_time
        avg_frame_time = np.mean(frame_times) * 1000 if frame_times else 0
        avg_process_time = np.mean(processing_times) * 1000 if processing_times else 0
        
        print(f"\nFinal Statistics:")
        print(f"Total frames processed: {frame_count}")
        print(f"Total runtime: {total_time:.2f} seconds")
        print(f"Average FPS: {frame_count/total_time:.2f}")
        print(f"Average frame time: {avg_frame_time:.1f}ms")
        print(f"Average processing time: {avg_process_time:.1f}ms")
        
        # Cleanup
        camera.release()
        cv2.destroyAllWindows()
        cleanup_ffmpeg_process(ffmpeg_process)
```

### Phase-Based Implementation
```python
class RealtimePipelineManager:
    """
    Manages different phases of real-time pipeline implementation.
    
    This class provides a framework for starting simple and adding
    complexity only when needed.
    """
    
    def __init__(self, width=640, height=480, fps=30):
        self.width = width
        self.height = height
        self.fps = fps
        self.phase = 1  # Start with simple approach
        
    def run_phase_1_simple(self):
        """Phase 1: Simple just-in-time processing."""
        print("Running Phase 1: Simple Real-Time Pipeline")
        return simple_realtime_pipeline()
        
    def run_phase_2_monitored(self):
        """Phase 2: Add performance monitoring."""
        print("Running Phase 2: Monitored Real-Time Pipeline")
        return monitored_realtime_pipeline()
        
    def run_phase_3_hybrid(self):
        """Phase 3: Hybrid approach with fallback to synchronization."""
        print("Running Phase 3: Hybrid Pipeline")
        return self.hybrid_pipeline()
        
    def hybrid_pipeline(self):
        """
        Hybrid pipeline that starts simple but can fall back to
        synchronization if performance issues are detected.
        """
        
        # Configuration
        WIDTH = self.width
        HEIGHT = self.height
        FPS = self.fps
        ALPHA = 0.7
        
        # Setup camera
        camera = cv2.VideoCapture(0)
        if not camera.isOpened():
            raise RuntimeError("Failed to open camera")
            
        camera.set(cv2.CAP_PROP_FRAME_WIDTH, WIDTH)
        camera.set(cv2.CAP_PROP_FRAME_HEIGHT, HEIGHT)
        camera.set(cv2.CAP_PROP_FPS, FPS)
        camera.set(cv2.CAP_PROP_BUFFERSIZE, 1)
        
        # Setup FFmpeg
        ffmpeg_process = setup_ffmpeg_process(WIDTH, HEIGHT, FPS, "hybrid_output.mp4")
        
        # Audio source setup
        # audio_source = setup_stm32_interface()
        
        # Performance monitoring
        frame_count = 0
        start_time = time.monotonic()
        processing_times = []
        use_sync = False  # Start without synchronization
        
        print("Hybrid Real-Time Heatmap Pipeline")
        print("Press 'q' to quit, 's' to toggle synchronization")
        
        try:
            while True:
                frame_start = time.monotonic()
                
                # Capture video frame
                ret, video_frame = camera.read()
                if not ret:
                    continue
                    
                # Get audio data
                audio_start = time.monotonic()
                # audio_data = audio_source.get_latest_fft_result()
                audio_data = generate_fake_heatmap(WIDTH, HEIGHT)
                audio_time = time.monotonic() - audio_start
                
                # Process
                process_start = time.monotonic()
                output_frame = apply_heatmap_overlay(audio_data, video_frame, ALPHA)
                process_time = time.monotonic() - process_start
                
                # Add status info to frame
                elapsed = time.monotonic() - start_time
                fps_actual = frame_count / elapsed if elapsed > 0 else 0
                
                cv2.putText(output_frame, f"FPS: {fps_actual:.1f}", 
                           (10, 30), cv2.FONT_HERSHEY_SIMPLEX, 0.7, (255, 255, 255), 2)
                cv2.putText(output_frame, f"Mode: {'Sync' if use_sync else 'Simple'}", 
                           (10, 60), cv2.FONT_HERSHEY_SIMPLEX, 0.7, (255, 255, 255), 2)
                cv2.putText(output_frame, f"Process: {(audio_time+process_time)*1000:.1f}ms", 
                           (10, 90), cv2.FONT_HERSHEY_SIMPLEX, 0.7, (255, 255, 255), 2)
                
                # Display and record
                cv2.imshow('Hybrid Heatmap', output_frame)
                send_to_ffmpeg(output_frame, ffmpeg_process)
                
                # Record timing data
                frame_count += 1
                processing_times.append(audio_time + process_time)
                
                # Check for performance issues and suggest sync
                if frame_count % 100 == 0:
                    avg_process_time = np.mean(processing_times[-100:]) * 1000
                    if avg_process_time > 33 and not use_sync:
                        print(f"Performance issue detected: {avg_process_time:.1f}ms processing time")
                        print("Consider enabling synchronization (press 's')")
                
                # Handle user input
                key = cv2.waitKey(1000 // FPS) & 0xFF
                if key == ord('q'):
                    print("Quit requested by user")
                    break
                elif key == ord('s'):
                    use_sync = not use_sync
                    print(f"Synchronization {'enabled' if use_sync else 'disabled'}")
                    
        except KeyboardInterrupt:
            print("\nPipeline interrupted by user")
        except Exception as e:
            print(f"Error in pipeline: {e}")
        finally:
            # Final statistics
            total_time = time.monotonic() - start_time
            avg_process_time = np.mean(processing_times) * 1000 if processing_times else 0
            
            print(f"\nFinal Statistics:")
            print(f"Total frames processed: {frame_count}")
            print(f"Total runtime: {total_time:.2f} seconds")
            print(f"Average FPS: {frame_count/total_time:.2f}")
            print(f"Average processing time: {avg_process_time:.1f}ms")
            print(f"Final mode: {'Synchronized' if use_sync else 'Simple'}")
            
            # Cleanup
            camera.release()
            cv2.destroyAllWindows()
            cleanup_ffmpeg_process(ffmpeg_process)
```

## 7. Performance Considerations

### Timing Requirements
- **Target frame rate**: 30 FPS = 33.33ms per frame
- **STM32 processing**: Must complete FFT + beamforming in < 33ms
- **Pi processing**: Must complete overlay + display in < 33ms
- **Total latency**: Should be < 100ms for real-time feel

### Memory Management
- **No buffering**: Process frames immediately
- **Minimal memory**: Only store current frame data
- **Garbage collection**: Let Python handle memory cleanup
- **Monitor usage**: Watch for memory leaks during long runs

### CPU Optimization
- **Single-threaded**: Keep it simple for now
- **CPU affinity**: Pin to specific cores if needed
- **Priority scheduling**: Use real-time scheduling if available
- **Profile code**: Identify bottlenecks with profiling tools

### Error Handling
- **Connection failures**: Handle STM32 disconnections
- **Frame drops**: Continue processing if frames are missed
- **Performance degradation**: Monitor and warn about issues
- **Graceful shutdown**: Clean up resources properly

## 8. Decision Framework

### When to Use Simple Approach
✅ **Use simple approach when**:
- STM32 processing is consistently fast (< 33ms)
- Timing is predictable and stable
- Latency requirements are not critical
- You want to minimize complexity
- Debugging and maintenance are priorities

### When to Add Synchronization
⚠️ **Consider synchronization when**:
- STM32 processing time varies significantly
- You notice dropped frames or stale data
- Latency requirements are critical
- You need to handle timing variations
- Performance monitoring shows issues

### Performance Thresholds
- **Processing time < 25ms**: Simple approach should work
- **Processing time 25-33ms**: Monitor closely, consider sync
- **Processing time > 33ms**: Definitely need synchronization
- **FPS < 90% of target**: Investigate and optimize
- **Frame drops > 1%**: Add error handling and sync

### Implementation Strategy
1. **Start simple**: Implement basic just-in-time approach
2. **Add monitoring**: Track performance metrics
3. **Test with real data**: Use actual STM32 audio data
4. **Optimize based on data**: Make decisions based on real performance
5. **Add complexity only if needed**: Don't over-engineer
